* looking-glass

=looking-glass= provides profunctor optics for Emacs Lisp.

It includes core optics (=iso=, =lens=, =prism=, =traversal=, =grate=), indexed variants,
keyed optics (=ix=, =at=), and convenience operations for viewing, updating, and folding.

* Architecture

The library is implemented as profunctor optics with interpreter-driven operations.

- Core optic representation is function-based: an optic transforms =p a b= into =p s t=.
- Reviews are interpreted through =Tagged= (no separate review slot on optics).
- Reversal uses =Re= interpretation.
- Folds/getters are interpreted via accumulation profunctors (=Const=/=Forget=-style behavior).
- Indexed optics use an explicit indexed wrapper value (=lg-indexed=), not ad-hoc bare callback plumbing.
- =lg-as-indexed= lifts unindexed optics with sentinel index =lg-no-index=.

Internal representation profunctors are exposed in code comments and naming:
=Exchange= (iso), =Shop= (lens), =Market= (prism), =Stall= (affine traversal),
=Bazaar= (traversal), =Forget= (fold/getter), =Tagged= (review), =Re= (reversal).

* Quickstart

#+begin_src elisp
(require 'looking-glass)
(require 'looking-glass-buffer)
#+end_src

** 1) Read and update nested pair fields

#+begin_src elisp
(setq pair-optic (lg-compose lg-car lg-cdr))

(lg-view pair-optic '((1 . 2) . 3))
;; => 2

(lg-set pair-optic 7 '((1 . 2) . 3))
;; => ((1 . 7) . 3)
#+end_src

** 1b) Compose optics (including variadic)

#+begin_src elisp
;; Composition order is outer-after-inner
(lg-view (lg-compose lg-car lg-cdr) '((1 . 2) . 3))
;; => 2

;; Changing order changes meaning
(lg-view (lg-compose lg-cdr lg-car) '((1 . 2) . 3))
;; => 3

;; Composition is variadic
(setq source '(:user (:profile [10 20 30])))
(lg-view (lg-compose (lg-ix :user) (lg-ix :profile) (lg-nth 1)) source)
;; => 20
#+end_src

** 2) Update all list items through a traversal

#+begin_src elisp
(lg-over lg-list (lambda (x) (* x 10)) '(1 2 3))
;; => (10 20 30)
#+end_src

** 3) Preview and construct with a prism

#+begin_src elisp
(setq non-nil lg-non-nil)

(lg-preview non-nil nil)
;; => lg-nothing

(lg-review non-nil 42)
;; => 42
#+end_src

** 3b) Negate booleans with a generic iso

#+begin_src elisp
(lg-view lg-not t)
;; => nil

(lg-view lg-not lg-true)
;; => lg-false

(lg-over lg-not #'identity lg-false)
;; => lg-true
#+end_src

** 4) Distinguish missing focus from present =nil=

#+begin_src elisp
(setq user '(:name nil :age 10))

(lg-preview (lg-ix :name) user)
;; => (lg-just)

(lg-preview (lg-ix :missing) user)
;; => lg-nothing
#+end_src

** 4b) Negative has checks use plain =not=

#+begin_src elisp
(setq profile '(:name "Ada" :age 10))

(lg-has (lg-ix :name) profile)
;; => t

(not (lg-has (lg-ix :missing) profile))
;; => t
#+end_src

** 5) Use =ix= for update-only-when-present

#+begin_src elisp
(lg-set (lg-ix :age) 11 '(:name "Ada" :age 10))
;; => (:name "Ada" :age 11)

(lg-set (lg-ix :missing) 11 '(:name "Ada" :age 10))
;; => (:name "Ada" :age 10)
#+end_src

** 6) Use =at= with tagged maybe (or compose =lg-unmaybe=)

#+begin_src elisp
(setq profile '(:name "Ada" :age 10))

(lg-view (lg-at :name) profile)
;; => (lg-just . "Ada")

(lg-set (lg-at :city) (lg-just "London") profile)
;; => (:name "Ada" :age 10 :city "London")

(lg-set (lg-at :age) lg-nothing profile)
;; => (:name "Ada")

(lg-view (lg-compose (lg-at :name) lg-unmaybe) profile)
;; => "Ada"
#+end_src

** 7) Indexed traversals expose index + value

#+begin_src elisp
(lg-ito-list-of lg-indexed-list '(10 11 12))
;; => ((0 . 10) (1 . 11) (2 . 12))

(lg-iover lg-indexed-list (lambda (i x) (+ i x)) '(10 11 12))
;; => (10 12 14)
#+end_src

** 7b) Fold focused values

#+begin_src elisp
;; Sum all focused values
(lg-foldl-of lg-list #'+ 0 '(1 2 3 4))
;; => 10

;; Indexed fold: weighted sum by index
(lg-ifoldl-of lg-indexed-list
              (lambda (acc i x)
                (+ acc (* i x)))
              0
              '(2 3 4))
;; => 11

;; Fold-map with a monoid
(lg-fold-map-of lg-list lg-monoid-string
                (lambda (x) (number-to-string x))
                '(1 2 3))
;; => "123"

;; Aggregate helpers
(lg-sum-of lg-list '(1 2 3 4))
;; => 10

(lg-maximum-of lg-list '(3 7 5))
;; => (lg-just . 7)
#+end_src

** 8) Conversion optics

#+begin_src elisp
(lg-preview lg-number-string "  -3.5 ")
;; => (lg-just . -3.5)

(lg-review lg-number-string 15)
;; => "15"
#+end_src

** 9) Update one list/string/vector position

#+begin_src elisp
(lg-over (lg-nth 2) (lambda (x) (* x 10)) '(4 5 6 7))
;; => (4 5 60 7)

(lg-set (lg-nth 1) ?Z "abcd")
;; => "aZcd"

(let ((bits (make-bool-vector 3 nil)))
  (aset bits 0 t)
  (lg-set (lg-nth 1) t bits))
;; => #&3"\7"
#+end_src

** 10) Filter by value or index

#+begin_src elisp
(setq even-only (lg-filtered (lambda (x) (and (numberp x) (zerop (% x 2))))))
(lg-over even-only (lambda (x) (* x 10)) 3)
;; => 3

(setq even-indexes (lg-indexed-list-indices (lambda (i) (zerop (% i 2)))))
(lg-iover even-indexes (lambda (_i x) (+ x 100)) '(10 11 12 13))
;; => (110 11 112 13)
#+end_src

** 11) Parse and edit JSON text with composed optics

#+begin_src elisp
(lg-over
 (lg-compose lg-json-parse (lg-ix "name"))
 #'upcase
 "{\"name\":\"Ada\",\"age\":10}")
;; => "{\"name\":\"ADA\",\"age\":10}"

(lg-over
 (lg-compose lg-json-parse (lg-ix "tags"))
 (lambda (tags)
   (let ((copy (copy-sequence tags)))
     (aset copy 1 (upcase (aref copy 1)))
     copy))
 "{\"tags\":[\"elisp\",\"optics\"]}")
;; => "{\"tags\":[\"elisp\",\"OPTICS\"]}"
#+end_src

** 12) Traverse JSON object values and members

#+begin_src elisp
(setq doc (lg-view lg-json-parse "{\"name\":\"Ada\",\"age\":10}"))

(gethash "name" doc)
;; => "Ada"

(gethash "age" doc)
;; => 10

(gethash "active" (lg-view lg-json-parse "{\"active\":true}"))
;; => lg-true
#+end_src

** 13) Use a grate for whole-structure rewrites

#+begin_src elisp
;; A 2x2 transpose built as a grate.
(setq transpose-2x2
      (lg-grate
       (lambda (build)
         (cons (cons (funcall build (lambda (s) (car (car s))))
                     (funcall build (lambda (s) (car (cdr s)))))
               (cons (funcall build (lambda (s) (cdr (car s))))
                     (funcall build (lambda (s) (cdr (cdr s)))))))))

(lg-over transpose-2x2 #'identity '((1 . 2) . (3 . 4)))
;; => ((1 . 3) . (2 . 4))

(lg-set transpose-2x2 0 '((1 . 2) . (3 . 4)))
;; => ((0 . 0) . (0 . 0))
#+end_src

** 14) Reverse a long prism+lens chain with =lg-re=

#+begin_src elisp
;; A long chain with mixed prism/lens pieces.
;; This shape uses the canonical lens-default context so reverse construction
;; can reproduce the full original structure exactly.

(setq id-optic
      (lg-compose
       lg-right-o
       lg-cdr
       lg-car
       lg-number-string))

(setq response
      '(right nil "42"))

(lg-view id-optic response)
;; => 42

;; Reverse the optic: now it builds envelope-shaped data from an id.
(setq id-optic-r (lg-re id-optic))

(lg-view id-optic-r 123)
;; => (right nil "123")

(equal (lg-view id-optic-r 42) response)
;; => t
#+end_src

** 15) Reversible parsed-JSON optic

#+begin_src elisp
;; Parse JSON, then parse the inner JSON string as a number.
;; This chain is reversible in both directions.

(setq json-number-optic
      (lg-compose lg-json-parse lg-number-string))

(lg-view json-number-optic "\"42\"")
;; => 42

(lg-review json-number-optic 123)
;; => "\"123\""

(setq json-number-optic-r (lg-re json-number-optic))

(lg-view json-number-optic-r 123)
;; => "\"123\""

(lg-review json-number-optic-r "\"9\"")
;; => 9
#+end_src

** 16) Real-world-ish regex optics examples

#+begin_src elisp
;; Load regex optics extension:
;; (require 'looking-glass-regex)

;; a) Normalize IDs in logs (all matches)
(setq log-line "request id=42 user=7 elapsed=153ms")
(lg-over (lg-regex-all "[0-9]+")
         (lambda (n) (format "[%s]" n))
         log-line)
;; => "request id=[42] user=[7] elapsed=[153]ms"

;; b) Rename only a captured value (first match)
(setq profile-line "name=Ada role=admin")
(lg-set (lg-regex-group "name=\\([A-Za-z]+\\)" 1)
        "Grace"
        profile-line)
;; => "name=Grace role=admin"

;; c) Increment every `id=` value in text
(setq payload "id=1 x id=22 y id=300")
(lg-over (lg-regex-all-groups "id=\\([0-9]+\\)" 1)
         (lambda (n) (number-to-string (1+ (string-to-number n))))
         payload)
;; => "id=2 x id=23 y id=301"

;; d) Redact email-like strings in free text
(setq note "contact: ada@example.com, ops@example.org")
(lg-regex-replace "[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]+"
                  (lambda (_m) "<redacted>")
                  note)
;; => "contact: <redacted>, <redacted>"
#+end_src

** 17) Word and line traversals

#+begin_src elisp
;; Requires: (require 'looking-glass-regex)

;; a) Word traversal (`\\w+` segments)
(lg-to-list-of lg-worded "Ada Lovelace 1843")
;; => ("Ada" "Lovelace" "1843")

(lg-over lg-worded #'downcase "Ada LOVELACE")
;; => "ada lovelace"

;; b) Custom word policy (here: Capitalized words only)
(let ((case-fold-search nil))
  (lg-to-list-of (lg-worded-regexp "[A-Z][a-z]+") "Ada LOVELACE Byron"))
;; => ("Ada" "Byron")

;; c) Line traversal preserving original separators
(setq mixed-lines (concat "alpha" "\r\n" "beta" "\u2028" "gamma" "\n"))

(lg-to-list-of lg-lined mixed-lines)
;; => ("alpha" "beta" "gamma" "")

(lg-over lg-lined #'upcase mixed-lines)
;; => "ALPHA\r\nBETA\u2028GAMMA\n"
#+end_src

** 18) Stateful buffer updates (monad + bang helpers)

#+begin_src elisp
;; Work with a live buffer through snapshot optics.
(with-temp-buffer
  (insert "hello world")
  (goto-char 1)
  (set-mark 6)
  (let* ((buffer (current-buffer))
         (program
          (lg-state-bind
           (lg-state-view-of lg-buffer-state-string)
           (lambda (before)
             (lg-state-bind
              (lg-state-over-of lg-buffer-state-region-string #'upcase)
              (lambda (_)
                (lg-state-bind
                 (lg-state-view-of lg-buffer-state-string)
                 (lambda (after)
                   (lg-state-pure (cons before after)))))))))))
    (lg-run-state-on-ref! program buffer)))
;; => ("hello world" . "HELLO world")

;; For direct mutation, use bang helpers over ref optics.
(with-temp-buffer
  (insert "alpha beta")
  (goto-char 1)
  (set-mark 6)
  (let ((buffer (current-buffer)))
    (lg-over! lg-buffer-state-region-string #'upcase buffer)
    (lg-set! lg-buffer-state-point 3 buffer)
    (buffer-string)))
;; => "ALPHA beta"

;; Reversible edit token.
(with-temp-buffer
  (insert "hello")
  (let* ((buffer (current-buffer))
         (undo (lg-ref-over-edit! lg-buffer-state-string
                                  (lambda (text) (concat text " world"))
                                  buffer)))
    (buffer-string)
    ;; => "hello world"
    (funcall undo)
    (buffer-string)))
;; => "hello"
#+end_src

* Main API groups

- Core constructors: =lg-iso=, =lg-lens=, =lg-prism=, =lg-traversal=, =lg-grate=, =lg-affine=
- Composition: =lg-compose=, =lg-compose-indexed=
- Sum helpers: =lg-just=/=lg-nothing= (Maybe), =lg-left=/=lg-right= (Either)
- Keyed optics: =lg-ix=, =lg-at=, =lg-plist-key=, =lg-alist-key=, =lg-hash-key=
- Sum-type optics: =lg-just-o=, =lg-left-o=, =lg-right-o=, =lg-non-nil=
- Boolean adapters: =lg-unbool=, =lg-not=
- Maybe adapters: =lg-unmaybe=
- Viewing and updates: =lg-view=, =lg-preview=, =lg-over=, =lg-set=
- Stateful refs/monad: =lg-state-pure=, =lg-state-bind=, =lg-state-ap=, =lg-state-over-of=, =lg-run-state-on-ref!=, =lg-over!=, =lg-set!=, =lg-ref-over-edit!=
- Folds: =lg-to-list-of=, =lg-foldl-of=, =lg-foldr-of=, =lg-fold-map-of=, =lg-first-of=, =lg-last-of=, =lg-find-of=, =lg-any-of=, =lg-all-of=, =lg-length-of=, =lg-sum-of=, =lg-product-of=, =lg-minimum-of=, =lg-maximum-of=
- Indexed folds and updates: =lg-ito-list-of=, =lg-ifoldl-of=, =lg-ifoldr-of=, =lg-ifold-map-of=, =lg-ifirst-of=, =lg-ilast-of=, =lg-ifind-of=, =lg-ilength-of=, =lg-isum-of=, =lg-iproduct-of=, =lg-iminimum-of=, =lg-imaximum-of=, =lg-iover=
- Review helpers: =lg-review=, =lg-unto=, =lg-reviews=
- Monoids: =lg-monoid-list=, =lg-monoid-string=, =lg-monoid-sum=, =lg-monoid-product=, =lg-monoid-any=, =lg-monoid-all=
- Generic extension points: =lg-ix-get=, =lg-ix-set=, =lg-at-get=, =lg-at-set=, =lg-nth-get=, =lg-nth-set=

- =looking-glass-regex=: regex/string optics (=lg-regex-match=, =lg-regex-group=, =lg-regex-all=, =lg-worded=, =lg-lined=, ...)

* Notes on semantics

- =lg-preview= returns tagged maybe: =lg-nothing= or =(lg-just . VALUE)=.
- =lg-at= focuses tagged maybe; setting =lg-nothing= removes and setting =(lg-just . VALUE)= inserts/updates.
- Compose with =lg-unmaybe= when you intentionally want nil-ambiguous shorthand behavior.
- =lg-compose= uses "outer after inner" ordering: =(lg-compose o1 o2 o3)= means run =o3= then =o2= then =o1=.
- Lifted unindexed traversals use =lg-no-index= as the sentinel index.
- =lg-re= supports reversals through profunctor dual machinery where available (notably =Strong/Costrong= and =Choice/Cochoice= paths).
